Recruit Got Owned:
A Full Chain CTF Breakdown

CTF Writeup  ·  4 min read  ·  Web Exploitation

The attacker didn't brute-force anything. They read source code through a misconfigured file fetcher, found plaintext credentials, and pivoted through SQL injection to extract the admin password. Three vulnerabilities, two flags, one clean chain.

The Attack Surface

The landing page FAQ reveals a CV fetch endpoint at /file.php?cv=<URL>. HTTP and HTTPS URLs are described as supported. A footer link to api.php becomes a critical pivot point. The filter blocks external URLs — but only checks that the value starts with file://.

Phase 1 — Source Enumeration via LFI

01
Read the fetcher source
GET /file.php?cv=file:///var/www/html/file.php

Filter confirmed: must start with file://, path resolved via realpath(), restricted to /var/www/html. Directory traversal blocked. Scope limited to webroot.

02
api.php — hardcoded credential found
GET /file.php?cv=file:///var/www/html/api.php
$HR_PASSWORD = '██████████████';

Plaintext HR password stored directly in PHP config. Labeled "temporary for initial rollout" — the classic antipattern that never gets cleaned up.

03
login.php — auth flow analysis
GET /file.php?cv=file:///var/www/html/login.php

HR login: direct string compare against $HR_PASSWORD, username hr. Admin login: prepared statement hits DB — password not in source.

04
dashboard.php — flag logic + SQLi identified
GET /file.php?cv=file:///var/www/html/dashboard.php
$query = "SELECT * FROM candidates WHERE name LIKE '%$search%'";

Role-based flag serving: hr → /user.txt · admin → /admin.txt. Unsanitized $search interpolated directly into raw MySQL query.

Vulnerability Summary

[01] SSRF / LFI via file:// scheme

The CV fetch endpoint accepted the file:// scheme with no restriction, enabling arbitrary local file reads within /var/www/html. No external URL validation present.

Allowlist trusted external domains only. Block file://, gopher://, and all non-HTTP schemes at the protocol level.
[02] Hardcoded credentials in source

Plaintext HR password stored in api.php, readable via the LFI. Intended as "temporary" — a common antipattern that persists indefinitely.

Use environment variables or a secrets manager. Never commit credentials to source files.
[03] Union-based SQL injection

The search GET parameter interpolated directly into a raw MySQL query. Enabled full table enumeration and credential extraction.

Use prepared statements with parameterized queries for all user-supplied input. Never interpolate raw strings into SQL.
The Chain Lesson: No single bug broke this app. It was the combination — LFI gave source, source gave creds, creds gave login, login gave SQLi context, SQLi gave escalation. Patch one link and the chain breaks.

Stay paranoid. Read source when you can. Rotate secrets when they leak. And always check the footer links.
